# ADR-0001: Use Phaser For Browser MVP

## Status
Accepted

## Context
Project ICE needs a browser-ready 2D renderer with input handling, sprite support, simple camera control, and fast iteration. The simulation itself must remain deterministic and testable without rendering.

## Decision
Use Phaser for the browser MVP presentation layer.

Phaser owns:
- scene lifecycle
- input capture
- sprites and simple animation
- camera and visual scaling
- HUD rendering hooks

Phaser does not own:
- rules
- authoritative game state
- deterministic physics
- AI decisions
- replay validation

## Consequences
- Rendering can be developed quickly without building a custom canvas stack.
- Simulation modules must expose plain data snapshots for Phaser to render.
- Automated rules, physics, replay, and AI tests must run without Phaser.
